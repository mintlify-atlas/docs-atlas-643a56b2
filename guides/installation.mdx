The midPilot Connector Generator is a FastAPI-based microservice for intelligent connector generation. This guide will walk you through the installation process.

## Prerequisites

Before installing the midPilot Connector Generator, ensure you have the following installed:

  <Card title="Python 3.12+" icon="python" href="https://www.python.org/downloads/">

  <Card title="UV Package Manager" icon="package" href="https://github.com/astral-sh/uv">
    Modern Python package manager
  </Card>
  <Card title="PostgreSQL 15" icon="database" href="https://www.postgresql.org/download/">

## Installation Methods

Choose your preferred installation method:

<Tabs>
  <Tab title="Using UV (Recommended)">
    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone <repository-url>
        cd connector-generator
        ```
      </Step>

      <Step title="Install UV package manager">
        If you haven't already installed UV:
        ```bash
        curl -LsSf https://astral.sh/uv/install.sh | sh
        ```
      </Step>

      <Step title="Install dependencies">
        UV will automatically create a virtual environment and install all dependencies:
        ```bash
        uv sync
        ```
        
        This installs all production dependencies listed in `pyproject.toml`.
      </Step>

      <Step title="Verify installation">
        Check that the installation was successful:
        ```bash
        uv run python --version
        # Should show Python 3.12 or later
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Docker">
    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone <repository-url>
        cd connector-generator
        ```
      </Step>

      <Step title="Build the Docker image">
        ```bash
        docker compose build
        ```
        
        This builds the image using the multi-stage Dockerfile, which:
        - Uses Python 3.13 base image
        - Installs dependencies with UV
        - Installs Playwright browsers for web scraping
      </Step>

      <Step title="Verify the build">
        ```bash
        docker images | grep midpilot-connector-gen
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Development Dependencies

For development, install additional tools for testing, linting, and type checking:

```bash
uv sync --group dev
```

This installs:
- **mypy** - Static type checker
- **pytest** - Testing framework
- **ruff** - Fast Python linter and formatter
- **pre-commit** - Git hook framework
- **poethepoet** - Task runner

## Installing Additional Dependencies

<AccordionGroup>
  <Accordion title="Production Dependencies">
    To add a production dependency:
    ```bash
    uv add <package-name>
    ```
    
    Example:
    ```bash
    uv add requests
    ```
  </Accordion>

  <Accordion title="Development Dependencies">
    To add a development dependency:
    ```bash
    uv add --dev <package-name>
    ```
    
    Example:
    ```bash
    uv add --dev pytest-cov
    ```
  </Accordion>
</AccordionGroup>

## Pre-commit Hooks

Set up pre-commit hooks to ensure code quality on every commit:

```bash
uv run pre-commit install
```

This installs hooks from `.pre-commit-config.yaml` that will automatically run quality checks (linting, formatting, type checking) before each commit.

## Troubleshooting

<AccordionGroup>
  <Accordion title="UV command not found">
    If `uv` is not found after installation, ensure that `~/.cargo/bin` is in your PATH:
    ```bash
    export PATH="$HOME/.cargo/bin:$PATH"
    ```
    
    Add this line to your `~/.bashrc` or `~/.zshrc` to make it permanent.
  </Accordion>

  <Accordion title="Python version mismatch">
    Ensure you have Python 3.12 or later:
    ```bash
    python --version
    ```
    
    UV will use the system Python by default. To use a specific version:
    ```bash
    uv python install 3.13
    uv python pin 3.13
    ```
  </Accordion>

  <Accordion title="Permission errors">
    If you encounter permission errors, avoid using `sudo`. Instead, ensure your user has write permissions to the project directory:
    ```bash
    chmod -R u+w .
    ```
  </Accordion>
</AccordionGroup>
